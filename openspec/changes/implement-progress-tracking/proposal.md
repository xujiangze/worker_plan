# 变更: 实现进度追踪功能

## 为什么
当前系统已经定义了进度追踪的规范(progress-tracking 和 progress-tracking-ui),但尚未实现该功能。用户需要能够更新计划状态、跟踪进度百分比、查看进度历史记录,以及获得到期提醒,以便更好地管理工作计划的执行情况。

## 变更内容
- 实现后端进度追踪 API:
  - 更新计划状态 API (PATCH /api/plans/:id/status)
  - 更新计划进度 API (PATCH /api/plans/:id/progress)
  - 状态转换验证逻辑
  - 进度与状态联动逻辑
  - 历史记录自动保存

- 实现前端进度追踪 UI:
  - 状态更新界面(下拉菜单)
  - 进度百分比输入功能
  - 进度条组件
  - 到期提醒功能
  - 加载状态和错误处理

## 影响
- 受影响的规范:
  - `progress-tracking` (实现现有规范)
  - `progress-tracking-ui` (实现现有规范)
- 受影响的代码:
  - 后端: API 路由、服务层、数据模型
  - 前端: 进度追踪页面、进度条组件、状态更新组件
- 受影响的数据库:
  - Plans 表(已有,需要确保字段完整)
  - PlanHistory 表(已有,需要确保字段完整)
