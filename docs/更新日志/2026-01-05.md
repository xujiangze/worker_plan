# 初始化项目
## 使用OpenSpec init
```bash
OpenSpec init
```

整体流程步骤:
1. 创建提议 `/openspec:proposal {具体提议(需求)}`
2. 审核规范 ``

## 让claude code 了解我的项目
claude 启动项目后对话框输入如下内容
```
1. Populate your project context:
   "Please read openspec/project.md and help me fill it out
    with details about my project, tech stack, and conventions"

2. Create your first change proposal:
   "I want to add [YOUR FEATURE HERE]. Please create an
    OpenSpec change proposal for this feature"

3. Learn the OpenSpec workflow:
   "Please explain the OpenSpec workflow from openspec/AGENTS.md
    and how I should work with you on this project"

注意: 始终用中文 
```

## 创建第一个提案(也就是一个新的需求)
指令: 在claude 下使用如下指令提交一个新的提案
```
/openspec:proposal 我需要完善一下需求, 原始的需求是 @README.md 将需求完善之后更新到 docs/需求/需求.md 
```
最终返回的案例如
```
  OpenSpec proposal 已通过严格验证:
  Change 'refine-requirements' is valid

  下一步

  现在你可以:
  1. 审查 proposal: 查看 openspec/changes/refine-requirements/proposal.md
  2. 查看需求文档: 查看 docs/需求/需求.md
  3. 批准后实施: 使用 /openspec:apply refine-requirements 开始实施
```

## 进行审查(review) [可用goland/vscode直接查阅, 不需要validate等]
**主要是针对 proposal 的内容看看有没有哪里不满足要求的**
按照上一步的输出, 我们可以
1. 审查 proposal: 查看 openspec/changes/refine-requirements/proposal.md
2. 查看 docs/需求/需求.md
3. 确认1,2步骤是否有需要修改, 如果有在命令行中给出建议
4. 如果3步骤不需要修改, 则执行 /openspec:apply refine-requirements

如果需要则使用
请修改 `refine-requirements` {具体建议}

## 自动实施
```
/openspec:apply refine-requirements
```
